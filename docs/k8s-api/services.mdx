# Services

`Service` — логический сервис с одним или несколькими транспортами (TCP/UDP/ICMP). Привязывается к `AddressGroup` через `ServiceBinding`. Используется в `Rule` через `endpoints.{local,remote}.type = Service`.

---

:::info Валидация и ошибки
Все валидации `spec` выполняются на уровне `sg-server` (gRPC-бэкенд); k8s-сервер
только проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::


## List

<ApiOperation method="GET" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/services">

Получить список ресурсов. Доступны две области: один namespace и весь кластер.

<Tabs groupId="scope">
  <TabItem value="namespace" label="В namespace" default>

<CodeBlock language="bash">
  {dedent`
    kubectl get services -n production
  `}
</CodeBlock>

  </TabItem>
  <TabItem value="cluster" label="По всему кластеру">

<CodeBlock language="bash">
  {dedent`
    kubectl get services -A
  `}
</CodeBlock>

  </TabItem>
</Tabs>

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl apply -f - <<EOF
    apiVersion: sgroups.io/v1alpha1
    kind: Service
    metadata:
      name: api
      namespace: production
    spec:
      displayName: API service
      transports:
        - protocol: TCP
          IPv: IPv4
          entries:
            - description: HTTP
              ports: "80,443"
        - protocol: UDP
          IPv: IPv4
          entries:

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/services/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч
- `application/merge-patch+json` — RFC 7396
- `application/json-patch+json` — RFC 6902

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl patch service api -n production --type=merge \\
      -p '{"spec":{"comment":"updated"}}'
  `}
</CodeBlock>
