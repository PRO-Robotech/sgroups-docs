# AddressGroups

`AddressGroup` — логическая группа адресов, к которой применяются `Rule`. IP появляются в группе через `HostBinding` и `NetworkBinding`.

---

:::info Валидация и ошибки
Все валидации `spec` производятся на уровне `sg-server` (gRPC-бэкенд) — k8s-сервер
лишь проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::

Получить список ресурсов. Доступны два scope: по namespace и по всему кластеру.

<Tabs groupId="scope">

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/addressgroups/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч
- `application/merge-patch+json` — RFC 7396
- `application/json-patch+json` — RFC 6902

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl patch addressgroup web -n production --type=merge \\
      -p '{"spec":{"defaultAction":"Allow"}}'
  `}
</CodeBlock>

## Delete

<ApiOperation method="DELETE" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/addressgroups/{name}">

Удалить объект по имени. Стандартные опции `--cascade`, `--grace-period`, `--timeout` поддерживаются как query-параметры.

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl delete addressgroup web -n production
  `}
</CodeBlock>

#### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "kind": "Status",
      "apiVersion": "v1",
      "status": "Success"
    }
    `}
</CodeBlock>

:::tip
Поле `refs[]` обновляется сервером автоматически. Изменять его в запросах `Create`/`Update` бессмысленно — оно будет перезаписано.
:::
