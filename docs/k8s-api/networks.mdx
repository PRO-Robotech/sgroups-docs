# Networks

`Network` — подсеть в формате CIDR. Привязывается к `AddressGroup` через `NetworkBinding` и попадает в nft-набор как элемент с флагом `interval`.

---

:::info Валидация и ошибки
Все валидации `spec` выполняются на уровне `sg-server` (gRPC-бэкенд); k8s-сервер
только проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::


## List

<ApiOperation method="GET" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/networks">

Получить список ресурсов. Доступны две области: один namespace и весь кластер.

<Tabs groupId="scope">
  <TabItem value="namespace" label="В namespace" default>

<CodeBlock language="bash">
  {dedent`
    kubectl get networks -n production
  `}
</CodeBlock>

  </TabItem>
  <TabItem value="cluster" label="По всему кластеру">

<CodeBlock language="bash">
  {dedent`
    kubectl get networks -A
  `}
</CodeBlock>

  </TabItem>
</Tabs>

#### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "kind": "NetworkList",

      "items": [

          "kind": "Network",

            "namespace": "production",
            "resourceVersion": "1110"
          },

            "CIDR": "10.0.0.0/24",
            "displayName": "Corporate /24"
          },
          "refs": []

    `}
</CodeBlock>

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/networks/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч
- `application/merge-patch+json` — RFC 7396
- `application/json-patch+json` — RFC 6902

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl patch network corp-net-10 -n production --type=merge \\
      -p '{"spec":{"CIDR":"10.0.0.0/16"}}'
  `}
</CodeBlock>
