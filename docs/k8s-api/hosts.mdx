# Hosts

`Host` — конкретный узел инфраструктуры. Поля `ips` и `metaInfo` заполняются агентом `sg-agent` при подключении и являются read-only.

---

:::info Валидация и ошибки
Все валидации `spec` производятся на уровне `sg-server` (gRPC-бэкенд) — k8s-сервер
лишь проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::


## List

<ApiOperation method="GET" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/hosts">

Получить список ресурсов. Доступны два scope: по namespace и по всему кластеру.

<Tabs groupId="scope">
  <TabItem value="namespace" label="В namespace" default>

<CodeBlock language="bash">
  {dedent`
    kubectl get hosts -n production
  `}
</CodeBlock>

  </TabItem>
  <TabItem value="cluster" label="По всему кластеру">

<CodeBlock language="bash">
  {dedent`
    kubectl get hosts -A
  `}
</CodeBlock>

  </TabItem>
</Tabs>

#### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "kind": "HostList",
      "apiVersion": "sgroups.io/v1alpha1",
      "items": [
        {
          "kind": "Host",
          "metadata": {
            "name": "host-01",
            "namespace": "production",
            "resourceVersion": "1120"
          },
          "spec": {
            "displayName": "Host 01"
          },
          "ips": {
            "IPv4": [
              "10.0.0.10"
            ],
            "IPv6": []
          },
          "metaInfo": {
            "hostName": "host-01.prod.local",
            "os": "linux",
            "platform": "ubuntu",
            "kernelVersion": "6.1.0"
          }
        }
      ]
    }
    `}
</CodeBlock>


<K8sValidation op="list" />

<K8sCodes codes={['unauthorized', 'forbidden', 'internal', 'serviceUnavailable']} />

</ApiOperation>

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/hosts/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч
- `application/merge-patch+json` — RFC 7396
- `application/json-patch+json` — RFC 6902

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl patch host host-01 -n production --type=merge \\
      -p '{"spec":{"description":"Updated"}}'
  `}
</CodeBlock>

## Delete

<ApiOperation method="DELETE" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/hosts/{name}">

Удалить объект по имени. Стандартные опции `--cascade`, `--grace-period`, `--timeout` поддерживаются как query-параметры.

  {dedent`
