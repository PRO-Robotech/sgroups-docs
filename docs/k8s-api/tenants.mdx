# Tenants

`Tenant` — **cluster-scoped**-ресурс, представляющий логическое пространство имен SGroups. Имя `Tenant` совпадает с именем namespace Kubernetes, в котором будут создаваться все namespaced-ресурсы.

---

:::info Валидация и ошибки
Все валидации `spec` выполняются на уровне `sg-server` (gRPC-бэкенд); k8s-сервер
только проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::

## Update (PUT)

<ApiOperation method="PUT" endpoint="/apis/sgroups.io/v1alpha1/tenants/{name}">

Полная замена объекта. Поле `metadata.resourceVersion` обязательно — оно защищает от race-condition (optimistic concurrency).

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    # kubectl edit делает GET → правки → PUT под капотом
    kubectl edit tenant production
  `}
</CodeBlock>


<K8sValidation op="put" serverItems={[

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/tenants/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч
- `application/merge-patch+json` — RFC 7396
- `application/json-patch+json` — RFC 6902

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl patch tenant production --type=merge \\
      -p '{"spec":{"description":"Updated"}}'
  `}
</CodeBlock>


<K8sValidation op="patch" serverItems={[
  { label: 'metadata.name', rules: RESTRICTIONS.name },
  { label: 'spec.displayName', rules: RESTRICTIONS.displayName },
  { label: 'spec.comment', rules: RESTRICTIONS.comment },
  { label: 'spec.description', rules: RESTRICTIONS.description },
]} />

<K8sCodes codes={['badRequest', 'unauthorized', 'forbidden', 'notFound', 'invalid', 'internal']} />

</ApiOperation>

## Delete

<ApiOperation method="DELETE" endpoint="/apis/sgroups.io/v1alpha1/tenants/{name}">

Удалить объект по имени. Стандартные опции `--cascade`, `--grace-period`, `--timeout` поддерживаются как query-параметры.

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl delete tenant production
  `}
</CodeBlock>

#### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "kind": "Status",
      "apiVersion": "v1",
      "status": "Success"
    }
    `}
</CodeBlock>

:::caution
Удаление тенанта каскадно удалит все namespaced-ресурсы (`AddressGroup`, `Rule`, и т.д.) в одноименном namespace.
:::


<K8sValidation op="delete" />

<K8sCodes codes={['unauthorized', 'forbidden', 'notFound', 'conflict', 'internal']} />

</ApiOperation>
