# Rules

`Rule` — правило межсетевого экрана. Эквивалент `UniRule` из REST API `sg-server`. Описывает разрешенный или запрещенный трафик между двумя `endpoints` (AG / Service / FQDN / CIDR).

---

:::info Валидация и ошибки
Все валидации `spec` выполняются на уровне `sg-server` (gRPC-бэкенд); k8s-сервер
только проксирует запрос. Формат ошибочных ответов и таблица кодов — в
[Обзоре API → Коды ошибок](/k8s-api/overview#коды-ошибок).
:::

#### Пример запроса

<CodeBlock language="bash">
  {dedent`
    kubectl apply -f - <<EOF
    apiVersion: sgroups.io/v1alpha1
    kind: Rule
    metadata:
      name: allow-web-to-db
      namespace: production
      annotations:
        linux-agent.sgroups.io/priority: "1000"
        linux-agent.sgroups.io/logs: "true"
    spec:
      action: Allow
      session:
        traffic: Egress
      endpoints:
        local:
          name: web
          namespace: production
          type: AddressGroup
        remote:
          name: db
          namespace: production
          type: AddressGroup
      transport:
        protocol: TCP
        IPv: IPv4
        entries:

    EOF

## Patch

<ApiOperation method="PATCH" endpoint="/apis/sgroups.io/v1alpha1/namespaces/{namespace}/rules/{name}">

Частичное обновление. Поддерживаемые `Content-Type`:

- `application/strategic-merge-patch+json` — стандартный k8s-патч

- `application/json-patch+json` — RFC 6902

  {dedent`
    kubectl patch rule allow-web-to-db -n production --type=merge \\

<CodeBlock language="bash">
