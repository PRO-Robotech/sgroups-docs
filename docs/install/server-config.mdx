# Конфигурация sg-server

`sg-server` использует `corlib/plain-config`. Источники конфигурации в порядке приоритета:

1. **Аргументы командной строки** — `--config <path>` указывает YAML-файл.
2. **Переменные окружения** с префиксом `SG_` (см. ниже).
3. **YAML-файл**.

В переменных окружения дефис `-` и слеш `/` в ключах заменяются на `_`:
ключ `server/graceful-shutdown` → `SG_SERVER_GRACEFUL_SHUTDOWN`.

## Полный пример YAML

<CodeBlock language="yaml">
  {dedent`
    logger:
      level: INFO

    metrics:
      enable: true

    healthcheck:
      enable: true

    server:
      endpoint: tcp://0.0.0.0:9006
      graceful-shutdown: 30s
      api-path-prefix: ""
      additional-api-paths: []
      use-proto-validator: true

    storage:
      type: postgres
      postgres:
        url: postgres://sgroups:secret@db:5432/sgroups?sslmode=disable

    authn:
      type: none
  `}
</CodeBlock>

При `authn/type = tls` сервер ожидает дополнительные параметры
(`authn/tls/cert-file`, `authn/tls/key-file`, `authn/tls/client/verify`,
`authn/tls/client/ca-files`). Подробности — в разделе
[Установка → Настройка TLS](/install/server-tls).

## Переменные окружения

Все ключи доступны через переменные окружения с префиксом `SG_`:

<CodeBlock language="bash">
  {dedent`
    export SG_SERVER_ENDPOINT="tcp://0.0.0.0:9006"
    export SG_LOGGER_LEVEL="INFO"
    export SG_STORAGE_POSTGRES_URL="postgres://sgroups:secret@localhost:5432/sgroups?sslmode=disable"
    export SG_METRICS_ENABLE="true"
  `}
</CodeBlock>

:::tip
В production рекомендуется хранить YAML-файл в репозитории и переопределять
чувствительные параметры (`SG_STORAGE_POSTGRES_URL`, ключи `authn`) через переменные окружения.
:::
