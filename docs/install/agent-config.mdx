# Конфигурация sg-agent

`sg-agent` использует `corlib/plain-config`. Источники конфигурации в порядке приоритета:

1. **Аргументы командной строки** — `--config <path>` указывает YAML-файл.
2. **Переменные окружения** с префиксом `NFT_` (см. ниже).
3. **YAML-файл**.

В переменных окружения дефис `-` и слеш `/` в ключах заменяются на `_`:
ключ `extapi/svc/sgroups/sync-status/interval` →
`NFT_EXTAPI_SVC_SGROUPS_SYNC_STATUS_INTERVAL`.

## Полный пример YAML

<CodeBlock language="yaml">
  {dedent`
    hostinfo:
      name: node-01
      namespace: prod

    exit-on-success: false
    continue-on-failure: true
    continue-after-timeout: 10s

    netns: ""
    def-policy-accept: false

    netlink:
      watcher:
        linger: 10s

    fqdn-rules:
      strategy: dns

    dns:
      nameservers: ["8.8.8.8"]
      proto: udp
      port: 53
      retries: 3
      retry-timeout: 1s
      dial-duration: 3s
      read-duration: 5s
      write-duration: 5s

    extapi:
      svc:
        def-daial-duration: 10s
        sgroups:
          address: tcp://sg-server:9006
          dial-duration: 3s
          sync-status:
            interval: 10s
            push: false
          use-json-codec: false
          authn:
            type: none

    telemetry:
      endpoint: 127.0.0.1:5000
      nft-collector:
        min-frequency: 10s
      metrics:
        enable: true
      healthcheck:
        enable: true
      profile:
        enable: true

    logger:
      level: INFO

    graceful-schutdown: 10s
  `}
</CodeBlock>

## Параметры: hostinfo

Идентификатор узла, по которому агент находит свой ресурс `Host` на сервере.
**Обязательные** параметры — без них агент не найдет себя в реестре.

<table>
  <thead>
    <tr>
      <th>Ключ YAML</th>
      <th>Env</th>
      <th>Default</th>
      <th>Описание</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>hostinfo/name</code></td>
      <td><code>NFT_HOSTINFO_NAME</code></td>
      <td>—</td>
      <td>Имя ресурса <code>Host</code> на сервере</td>
    </tr>
    <tr>
      <td><code>hostinfo/namespace</code></td>
      <td><code>NFT_HOSTINFO_NAMESPACE</code></td>
      <td>—</td>
      <td>Namespace ресурса <code>Host</code></td>
    </tr>
  </tbody>
</table>

## Переменные окружения

Все ключи доступны через переменные окружения с префиксом `NFT_`:

<CodeBlock language="bash">
  {dedent`
    export NFT_HOSTINFO_NAME="node-01"
    export NFT_HOSTINFO_NAMESPACE="prod"
    export NFT_EXTAPI_SVC_SGROUPS_ADDRESS="tcp://sg-server:9006"
    export NFT_LOGGER_LEVEL="INFO"
  `}
</CodeBlock>

:::tip
Минимальный набор для production — `hostinfo/{name,namespace}` и
`extapi/svc/sgroups/address`. Остальное в большинстве сценариев работает на
значениях по умолчанию.
:::
