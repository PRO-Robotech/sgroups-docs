# Аннотации `linux-agent.sgroups.io/*`

Аннотации правил — это сигналы для агента, которые **не меняют логику матча**, но добавляют
nft-модификаторы в готовую строку. Все они живут в `metadata.annotations` правила
(`/v1/rules/upsert`).

## Поддерживаемые ключи

| Аннотация | Тип | Эффект в nft |
|---|---|---|
| `linux-agent.sgroups.io/trace` | `"true"` / `"false"` | вставка `meta nftrace set 1` в строку |
| `linux-agent.sgroups.io/logs` | `"true"` / `"false"` | вставка `log level debug` в строку |
| `linux-agent.sgroups.io/priority` | целое число (строкой) | задает порядок строк внутри per-AG цепочки |

Любая отсутствующая или равная `"false"` аннотация модификатор не вставляет.

## Куда они попадают в строке

Базовый пример для всех табов ниже:

- local: `prod/web` (AG)
- remote: `prod/db` (AG)
- traffic: `INGRESS`, action: `ALLOW`
- транспорт: TCP/80

Меняется только содержимое `metadata.annotations` — посмотрите, как это отражается
в nft-строке. `accept`/`drop` всегда стоит **последним токеном** строки.

## priority — порядок строк

`priority` управляет позицией строки в per-AG цепочке. Чем меньше число — тем выше
строка. Агент сортирует строки в цепочке по возрастанию priority. Если две строки
имеют одинаковый priority — порядок не детерминирован.

:::caution Инверсия priority — баг
Если в одной per-AG цепочке строка с `priority=10100` оказалась **выше** строки с
`priority=10050` — это инверсия. Это значит, что агент применил правила в
неправильном порядке либо неправильно их пересортировал.
:::

## Что НЕ влияет на nft-строку

- `linux-agent.sgroups.io/priority` сам по себе **не добавляет токенов в строку** —
  только меняет её позицию в цепочке.
- Аннотация `all` (`logs+trace+priority`) с точки зрения nft-модификаторов
  эквивалентна сочетанию `logs+trace`.

## Просмотр трейса и логов

При `trace=true` пакеты, попавшие в правило, можно смотреть через nft-трейсер:

```bash
sudo nft monitor trace
```

При `logs=true` правило пишет debug-уровень в kernel log:

```bash
sudo dmesg -w | grep sgroups
# или через journald
journalctl -kf --grep sgroups
```
