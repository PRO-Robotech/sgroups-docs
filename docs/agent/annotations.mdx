# Аннотации `linux-agent.sgroups.io/*`

Аннотации правил — это сигналы для агента, которые **не меняют логику совпадения**, но добавляют
модификаторы nft в готовую строку. Все они задаются в `metadata.annotations` правила
(`/v1/rules/upsert`).

## Поддерживаемые ключи

| Аннотация | Тип | Эффект в nft |
|---|---|---|
| `linux-agent.sgroups.io/trace` | `"true"` / `"false"` | вставка `meta nftrace set 1` в строку |
| `linux-agent.sgroups.io/logs` | `"true"` / `"false"` | вставка `log level debug` в строку |
| `linux-agent.sgroups.io/priority` | целое число (строкой) | задает порядок строк внутри per-AG цепочки |

Если аннотация отсутствует или равна `"false"`, модификатор не добавляется.

## Куда они попадают в строке

Базовый пример для всех вкладок ниже:

- local: `prod/web` (AG);
- remote: `prod/db` (AG);
- traffic: `INGRESS`, action: `ALLOW`;
- транспорт: TCP/80.

Меняется только содержимое `metadata.annotations` — посмотрите, как это отражается
в строке nft. `accept` и `drop` всегда стоят **последним токеном** в строке.

## priority — порядок строк

`priority` управляет позицией строки в per-AG цепочке. Чем меньше число — тем выше
строка. Агент сортирует строки в цепочке по возрастанию `priority`. Если у двух строк
одинаковый `priority`, их относительный порядок не детерминирован.

:::caution Инверсия priority — баг
Если в одной per-AG цепочке строка с `priority=10100` оказалась **выше** строки с
`priority=10050` — это инверсия. Она означает, что агент применил правила в
неправильном порядке либо некорректно их пересортировал.
:::

## Что НЕ влияет на строку nft

- `linux-agent.sgroups.io/priority` сам по себе **не добавляет токенов в строку** —
  только меняет ее позицию в цепочке.
- Сочетание `logs+trace+priority` с точки зрения модификаторов nft
  эквивалентно сочетанию `logs+trace`.

## Просмотр трейсинга и логов

При `trace=true` пакеты, попавшие в правило, можно смотреть через трейсер nft:

```bash
sudo nft monitor trace
```

При `logs=true` правило пишет в журнал ядра на уровне debug:

```bash
sudo dmesg -w | grep sgroups
# либо через journald
journalctl -kf --grep sgroups
```
