## Upsert

<ApiOperation method="POST" endpoint="/v1/rules/upsert">

Создает новое правило или обновляет существующее. Тип правила определяется комбинацией `endpoints.local.type`,
`endpoints.remote.type` и `transport.protocol` (oneOf). Поддерживаются только комбинации, у которых есть
отдельная rule-таблица в БД: `ADDRESS_GROUP → ADDRESS_GROUP`, `ADDRESS_GROUP → FQDN`, `ADDRESS_GROUP → CIDR`,
локальное `ADDRESS_GROUP → ICMP`, `ADDRESS_GROUP → SERVICE`, `SERVICE → ADDRESS_GROUP`,
`SERVICE → SERVICE`, `SERVICE → FQDN`, `SERVICE → CIDR`.

:::info Асимметричные пары AG ↔ SERVICE
Для `ADDRESS_GROUP → SERVICE` и `SERVICE → ADDRESS_GROUP` режим `spec.transport`
зависит от направления трафика — на стороне, где трафик **направлен на Service**,
транспортная конфигурация наследуется из самого `Service` (блок `spec.transport`
не указывается). На стороне, где Service **источник** трафика, `spec.transport`
обязателен. Полная матрица — в табе **matrix** ниже.
:::

## Delete

<ApiOperation method="POST" endpoint="/v1/rules/delete">

Удаляет правило по `uid` или по комбинации `name` + `namespace`. Операция общая для всех типов правил.
