Обе стороны — сервисы (`SERVICE`). Транспортная конфигурация берется из ресурсов Service обеих сторон — блок `transport` **не указывается**.

:::warning Transport
Блок `spec.transport` **не указывается** при любом направлении трафика. Протоколы, порты и ICMP-типы определяются полем `transports` ресурсов Service, на которые ссылаются `local` и `remote`.
:::
