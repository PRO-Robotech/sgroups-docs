Локальная сторона — адресная группа (`ADDRESS_GROUP`), удалённая — сервис (`SERVICE`).
Поведение `transport` зависит от направления трафика:

- **Egress / Both** — `transport` НЕ указывается. Транспортная конфигурация берётся из ресурса Service (remote).
- **Ingress** — `transport` ОБЯЗАТЕЛЕН, задаётся явно в правиле.

Блок `spec.transport` отсутствует — транспортная конфигурация (протоколы, порты, ICMP-типы) наследуется из поля `transports` ресурса Service, на который ссылается `remote`.
