Локальная сторона — сервис (`SERVICE`), удалённая — IP-подсеть (`CIDR`).
Поле `remote.value` **обязательно**. Поведение `transport` зависит от направления трафика:

:::warning Transport и Traffic
- **Egress** — блок `spec.transport` **обязателен**, задаётся явно. Service отправляет трафик к CIDR.
- **Ingress** — блок `spec.transport` **не указывается**. Транспортная конфигурация наследуется из `transports` ресурса Service (local). CIDR отправляет трафик к Service.
:::
