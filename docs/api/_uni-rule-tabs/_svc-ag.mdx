Локальная сторона — сервис (`SERVICE`), удаленная — адресная группа (`ADDRESS_GROUP`).
Поведение `transport` зависит от направления трафика:

:::warning Transport
- **Egress / Both** — блок `spec.transport` **обязателен**, задаётся явно в правиле.
- **Ingress** — блок `spec.transport` не указывается. Транспортная конфигурация наследуется из `transports` ресурса Service (local).
:::
