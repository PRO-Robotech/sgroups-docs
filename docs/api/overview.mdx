# Обзор API

SGroups предоставляет единый API для управления сетевыми группами безопасности. API доступен по двум транспортам:

- **gRPC** — основной протокол, определенный через Protocol Buffers
- **REST** — автоматически сгенерированный через gRPC-Gateway, доступен по префиксу `/v1/*`

## Базовый URL

<CodeBlock>
  {dedent`
    # REST (gRPC-Gateway)
    http://localhost:9006/v1/

    # gRPC
    localhost:9000
  `}
</CodeBlock>

## Аутентификация

Все запросы к API должны содержать заголовок аутентификации. Конкретный механизм аутентификации зависит от конфигурации развертывания.

<CodeBlock language="bash">
  {dedent`
    curl -H 'Authorization: Bearer <token>' http://localhost:9006/v1/namespaces/list
  `}
</CodeBlock>

## Стандартные операции

Для каждого ресурса доступен набор стандартных операций:

### Upsert (создание / обновление)

<span className="api-method api-method--post">POST</span> <code className="api-endpoint">/v1/&#123;resource&#125;/upsert</code>

Создает ресурс или обновляет существующий. Если ресурс с указанным `name` и `namespace` уже существует — он будет обновлен.

### Delete (удаление)

<span className="api-method api-method--post">POST</span> <code className="api-endpoint">/v1/&#123;resource&#125;/delete</code>

Удаляет ресурс по `uid` или комбинации `name` + `namespace`.

### List (получение списка)

<span className="api-method api-method--post">POST</span> <code className="api-endpoint">/v1/&#123;resource&#125;/list</code>

Возвращает список ресурсов с фильтрацией через селекторы.

### Watch (наблюдение за изменениями)

<span className="api-method api-method--post">POST</span> <code className="api-endpoint">/v1/&#123;resource&#125;/watch</code>

Серверный стриминг (server-side streaming) — возвращает поток изменений ресурсов в реальном времени.

## Селекторы

Операции List и Watch поддерживают фильтрацию через селекторы.

:::info Логика комбинирования селекторов
- **Внутри одного селектора** `fieldSelector` и `labelSelector` объединяются по **И** (AND) — ресурс должен соответствовать обоим условиям.
- **Между селекторами** в массиве `selectors[]` действует логическое **ИЛИ** (OR) — ресурс попадает в результат, если соответствует хотя бы одному селектору.
:::

## Watch и resourceVersion

Механизм Watch позволяет получать изменения ресурсов в реальном времени:

1. Клиент отправляет запрос Watch с опциональным `resourceVersion`
2. Если `resourceVersion` указан — сервер отправляет все изменения, произошедшие после этой версии
3. Если не указан — сервер начинает с текущего состояния и далее отправляет изменения
4. Каждое событие содержит обновлённый `resourceVersion` для отслеживания позиции

<CodeBlock language="json">
  {dedent`
    {
      "resource": {
        "metadata": {
          "resourceVersion": "42"
        }
      }
    }
  `}
</CodeBlock>
