# Status API

API для получения информации о состоянии синхронизации SGroups.

---

## Получение статуса

<span className="api-method api-method--get">GET</span> <code className="api-endpoint">/v1/status/list</code>

Возвращает время последнего успешно завершенного обновления БД.

### Пример запроса

<CodeBlock language="bash">
  {dedent`
    curl -X GET http://localhost:9006/v1/status/list
  `}
</CodeBlock>

### Ответ

<table>
  <thead>
    <tr>
      <th>Параметр</th>
      <th>Тип</th>
      <th>Описание</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>updatedAt</code></td>
      <td><code>{TYPES.timestamp}</code></td>
      <td>Время последнего успешно завершенного обновления БД.</td>
    </tr>
  </tbody>
</table>

### Пример ответа

<CodeBlock language="json">
  {dedent`
    {
      "updatedAt": "2026-04-25T21:20:03.082663Z"
    }
  `}
</CodeBlock>

<Codes codes={['internal']} />

## Наблюдение за статусом

<span className="api-method api-method--get">GET</span> <code className="api-endpoint">/v1/status/watch</code>

Серверный стриминг статуса. В REST/gRPC-Gateway каждое событие приходит в
обёртке `result`.

### Пример запроса

<CodeBlock language="bash">
  {dedent`
    curl -N -X GET http://localhost:9006/v1/status/watch
  `}
</CodeBlock>

### Пример события

<CodeBlock language="json">
  {dedent`
    {
      "result": {
        "updatedAt": "2026-04-25T21:20:03.082663Z"
      }
    }
  `}
</CodeBlock>

POST-варианты `/v1/status/list` и `/v1/status/watch` не поддерживаются и
возвращают `501 Not Implemented`.

<Codes codes={['internal', 'unimplemented']} />
